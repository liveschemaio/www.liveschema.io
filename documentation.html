LiveScheme Doc

The LiveSchema could be installed on-site in customer private network. Here's a short notice of LiveSchema documentation avaliable by request. For software as a service use please contact us with email you will find in the bottom of the page.

Roles and access permisions

There are 2 main roles of LiveSchema user interface.

Designer
Watcher
Designer - a user who is able to manage all the objects, links and properties.

Watcher - a user who is able to watch and able to change anything.

Each of the Designers and Watchers can act in domain granted. So some of Designers can create/change/delete objects and links in A-domain, some of them can manage B-domain and some of them can manage both A and B domains. The same idea is for Watchers.

An Admin user can grand pemisions for users for the domains and grand permisions.

Another level of access is system console and superuser. The superuser permisions required for installations and global reconfiguration issues.

On-stie Deployment Requirements

The system can be installed on both hardware servers and virtual machines.

The recommended requirements for a server of the system are:

64-bit OS
CPU — 2.4 GHz / quad core
RAM — 4 GB
HDD — 72 GB

64-bit OS for the server of the system is required. The following operating systems were successfully tested:

Red Hat Enterprise Linux 5.5+ / 6 / 7;
SUSE Linux Enterprise 11 / 12;
Ubuntu Linux 10.04 / 12.04 / 14.04 / 14.10.

The server can be deployed with as a set of Docker Containers, Virtual or Hardware Machines meeting customer infrastructure design.

Interface settings

To change system title in web interface:
create a copy of default settings file (if not created earlier):
cp /usr/local/liveschema/client/js/default-client-config.js /usr/local/liveschema/client/js/client-config.js
enter new title to the custom settings file (/usr/local/liveschema/client/js/client-config.js) as follows: title: ‘<your-new-name>’

To change LiveSchema color-map for all the system you need to change /usr/local/liveschema/client/livescema.local/css/livescema.local.css file. We can provide you with a set of color schemas or prepare an individual for you. The file change is superuser privilege.

Note that you can style each of the object/elemnt individualy. There's a background image upload user interface control in each of the object properties. And you can style each of the object with custom pease of CSS through user interface control in each of the object properties. All of the user interface properties changes is available for Designer roles users only.

/usr/local/liveschema/client/livescema.local/css/livescema.local.css is customisation example is follows:


	.state-{ID} {
		-moz-box-shadow: 2px 5px 10px {COLOR};   /* Object shadow color */
		-webkit-box-shadow: 2px 5px 10px {COLOR};   /* Object shadow color */
		box-shadow: 2px 5px 10px {COLOR};   /* Object shadow color */
		border-left: 5px solid {COLOR}; /* Object main color */
	}

	.state-bg-{ID} {
		background-color: {COLOR}; /* Object background color */
	}

	.state-bg-light-{ID} {
		background-color: {COLOR}; /* Object background hue */
	}

	.view-screen-element ul li.child-object.state-{ID} {  /* Children object color */
		border-left: 5px solid {COLOR};
	}

	.badge.state-{ID} { /* Children badge color */
		background-color: {COLOR};
	}

	.link-state-{ID} circle { /* Link connector color */
		fill: {COLOR};
	}

	.link-state-{ID} path { /* Link color */
		stroke: {COLOR};
	}

where:


	{ID} - state ID;
	{COLOR} - colors of object and link elements.

Methods of control

There are a number of build in methods to control state of each object or link.

Process by name
Database query
SNMP Get
SNMP Trap
Program / script execution
Ping
Local port
Remote port
HTTP-request
Configuration file / directory
JMX-sensor
FTP-sensor
MQTT-sensor
Binary protocol

The methods allow you to control an almost any service, application or network infrastrucure. For each of particular object or link Designer is able to set of "what is good and what is bad" condition in web user interface.

Application Programming Interface

LiveSchema API allows to create/change/delete objects and links, change it's poperties and assotiated documents, change state to make scheme color change.

The system provides RESTful API to access to its functionality. API includes authentication, actions on Objects, Links and Classes and other functions.

The documents describing all of the REST methods, their parameters and format of responses are available for customers.

API uses HTTP 2.0, both unsecure HTTP and secure HTTPS (over SLL).

Associated documentation and properties

There's a functionality allowing Designers to add documentation and properties descriptions to an element context. That's meant that each and any object or link could have assotiated wiki-link or pdf-uploaded and the document will be accessible for Watchers. Designer can set an option with a text to a link or an object. The text will be shown in the object/link properties to the Watcher. Ther're an option to map the text right to the scheme.

As an example, you can map addresses of your elements on the LiveScheme. Those addresses will be shown right under the elements.
