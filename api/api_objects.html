OBJECTS

node/api/objects

GET

Возвращает список всех Объектов в системе, БЕЗ Свойств, в виде массива JSON-документов. Каждый JSON-документ содержит поля соответствующего Объекта, такие как id, name, class_id:

Body ответа сервера - массив Объектов:

[
{
    "id": <ID Объекта>,
    "parent_id": <ID родителя Объекта>,
    "class_id": <ID Класса Объекта>,
    "name": <Имя Объекта>,
    "client_data": {
        <Параметры отображения Объекта>
    },
    "background": <Название фонового изображения Объекта>,
    "geoposition": [
        <X координата геопозиции>,
        <Y координата геопозиции>
    ],
    "child_ref_ids": [
        <ID вложенной Ссылки 1>,
        <ID вложенной Ссылки 2>,
        ...
    ],
    "child_ids": [
        <ID вложенного Объекта 1>,
        <ID вложенного Объекта 2>,
        ...
    ],
    "child_link_ids": [
        <ID Связи вложенных Объектов 1>,
        <ID Связи вложенных Объектов 2>,
        ...
    ],
    "properties": [
        <Свойство 1>,
        <Свойство 2>,
        ...
    ]
    "state_id": <ID состояния Объекта>
},
...
]
POST

Создает новый Объект.

JSON-документ запроса:

{
    "name": <имя>,
    "class_id": <ID класса>
}

Значения по-умолчанию:

parent_id: 1
state_id: 1

Body ответа сервера:

{
  "parent_id": <ID родителя>,
  "name": <имя>,
  "class_id": <ID класса>,
  "geoposition": [],
  "child_ref_ids": [],
  "child_link_ids": [],
  "child_ids": [],
  "properties": [],
  "state_id": <ID состояния>,
  "id": <ID Объекта>
}

/node/api/objects/:id/stat

GET

Возвращает данные от Агента по Объекту с указанным ID в виде JSON-документа.

В поле payload содержится JSON-документ с полезными данными от Агента. Список полей в этом JSON-документе варьируется в зависимости от настроек мониторинга.

Body ответа сервера:

{
  "entityId": <ID Объекта>,
  "entityType": "obj",
  "taskType": <Тип проверки>,
  "period": <Период проверки>,
  "timestamp": <Таймстэмп создания проверки>,
  "agentId": <ID агента>,
  "agentVersion": <Версия агента>,
  "agentBuild": <Версия билда агента>,
  "payload": {
      <Данные от Агента 1>,
      <Данные от Агента 2>,
      ...
  }
}
POST

Принимает JSON-документ, переданный в теле запроса, в качестве данных от агента. Используется в случае, когда нужно осуществлять мониторинг средствами стороннего ПО.

Данные от агента должны иметь следующий формат:

{
  "entityId": "ID Объекта",
  "entityType": "obj",
  "period": "Период проверки",
  "payload": {...}
}

Поле payload содержит измерения по Объекту в формате JSON-документа. Это может быть либо одиночный JSON-документ, либо массив JSON-докуметов. Допустимы также вложенные документы.

Body ответа сервера:

  ...

/node/api/objects/:id/props

POST

Создает свойство для Объекта.

JSON-документ запроса:

{
  "name": <Название свойства>,
  "type_id": <Тип свойства>,
  "value": <Значение свойства>
}

Body ответа сервера:

{ "id": }

Доступны следующие типы свойств:

  • 1: Информация
  • 6: Ссылка
  • 7: Документ
  • 8: Сервис(Свойства проверки)

  "name": <Название свойства>,

/node/api/objects/:id/state-conditions

GET

Возвращает список условий смены состояний Объекта с указанным ID в виде массива JSON-документов.

Body ответа сервера - массив JSON-документов

[
  {
    "condition": {"Имя Метрики": { "Оператор": "Значение" }},
    "state": "ID Статуса"
  },
  ...
]

Возможные операторы:

"_eq" - "равно"
"_neq" - "не равно"
"_lt" - "меньше"
"_lte" - "меньше или равно"
"_gt" - "больше"
"_gte" - "больше или равно"
"_m" - "совпадает с"

Пример запроса

PUT

Модифицирует существующее условие перехода состояний Объекта.

Подробнее в примере запроса

DELETE

Удаляет существующее условие перехода состояний Объекта.

/node/api/objects/:id/state-conditions/exists

GET

Возвращает "true", если Объект с указанным ID содержит условия перехода состояний, "false" иначе.
